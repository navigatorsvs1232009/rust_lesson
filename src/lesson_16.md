16 УРОК ПО RUST: Документация, которую не испортит даже Майкл

## Как написать `///`, чтобы Пэм поняла, а Дуайт не стал её переписывать под «систему безопасности уровня Стрелок»

> *«Если документация не написана — это как провести тренинг по пожарной безопасности… без огнетушителя. И без пожара. И без тренинга.»*
> — Майкл Скотт, не зная, что он только что описал большинство README на GitHub

В Dunder Mifflin есть **одна неразрешимая проблема**, хуже даже принтера с призраком:

> **Никто не читает документацию.**
> …Пока не поздно.

А всё потому, что большинство доков — это:

- «Очевидно!» (Майкл)
- Страница из технического регламента (Дуайт)
- Пусто (Кевин)

Но в Rust **документация — часть кода**.
Она **компилируется**, **тестируется**, и **помогает новичкам не уйти в панику** при виде `Result<T, E>`.

Сегодня вы научитесь писать документацию **по правилам Rust**, но в офисном духе:

- Просто
- Понятно
- С примерами, которые можно скопировать и запустить
- И без упоминания облака (если только Майкл не начал)

Поехали!
